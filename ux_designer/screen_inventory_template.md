# Screen Inventory

## Screen Entry Format

Every screen must be documented as follows:

```
### Screen: [Screen Name]
**ID**: SCR-[Number]
**Related Flow**: [Flow Name]
**Related Stories**: US-XXX-XX

#### Purpose
What is this screen for? What does the user accomplish here?

#### Layout Description
Describe the visual layout from top to bottom:
- **Header**: [Navigation bar, back button, title, etc.]
- **Body**: [Main content area â€” lists, forms, cards, etc.]
- **Footer**: [Tab bar, action buttons, etc.]

#### Elements
| Element | Type | Behavior | Required |
|---|---|---|---|
| [Element name] | Button / Input / Text / Image / List / Card | [What happens on interaction] | Yes/No |

#### States
- **Default**: [What the screen looks like on first load]
- **Loading**: [What the user sees while data loads]
- **Empty**: [What the user sees if there's no data]
- **Error**: [What the user sees if something fails]
- **Success**: [What the user sees after successful action]

#### Navigation
- **Comes from**: [Screen Name] via [action]
- **Goes to**: [Screen Name] via [action]
```
